<p>LIBSVM Accelerated with GPU using the CUDA Framework</p><!--break--><p>&nbsp;</p><p><u>DESCRIPTION</u></p><p>GPU-accelerated LIBSVM is a modification of the <a href="http://www.csie.ntu.edu.tw/~cjlin/libsvm/">original LIBSVM</a> that exploits the CUDA framework to significantly reduce processing time while producing identical results. The functionality and interface of LIBSVM remains the same. The modifications were done in the kernel computation, that is now performed using the GPU.</p><p>Watch a short video on the capabilities of the GPU-accelerated LIBSVM package <a href="http://www.youtube.com/watch?v=Fl99tQQd55U">here</a></p><p>Change log &amp; full code at <a href="http://github.com/MKLab-ITI/CUDA">github</a></p><p>&nbsp;</p><p><u>FEATURES</u></p><p>Mode Supported</p><ul><li>C-SVC classification with RBF kernel</li></ul><p>Functionality / User interface</p><ul><li>Same as LIBSVM</li></ul><p>&nbsp;</p><p><u>PREREQUISITES</u></p><ul><li>LIBSVM prerequisites</li><li>NVIDIA Graphics card with CUDA support</li><li>Latest NVIDIA drivers for GPU</li></ul><p>&nbsp;</p><p><span style="text-decoration: underline;">PERFORMANCE COMPARISON</span></p><p>To showcase the performance gain using the GPU-accelerated LIBSVM we present an example run.</p><p>PC Setup</p><ul><li>Quad core Intel Q6600 processor</li><li>3.5GB of DDR2 RAM</li><li>Windows-XP 32-bit OS</li></ul><p>Input Data</p><ul><li>TRECVID 2007 Dataset for the detection of high level features in video shots</li><li>Training vectors with a dimension of 6000</li><li>20 different feature models with a variable number of input training vectors ranging from 36 up to 3772</li></ul><p>Classification parameters</p><ul><li>c-svc</li><li>RBF kernel</li><li>Parameter optimization using the easy.py script provided by LIBSVM.</li><li>4 local workers</li></ul><p>&nbsp;</p><p><img alt="" src="https://mklab.iti.gr/files/GPULIBSVM-comparison.jpg" style="width: 505px; height: 309px;" /></p><p>&nbsp;</p><p>Discussion</p><ul><li>GPU-accelerated LIBSVM gives a performance gain depending on the size of input data set. This gain is increasing dramatically with the size of the dataset.</li><li>Please take into consideration input data size limitations that can occur from the memory capacity of the graphics card that is used.</li></ul><p>&nbsp;</p><p><u>DOWNLOADS</u></p><p><strong>Version 1.2</strong></p><p><em>The latest version is based on LIBSVM version 3.17 &amp; Everybody is welcome to make the port and we can host the ported software.</p><p>&nbsp;</p><p><strong><span style="font-size: 14px;">2. Visual Studio will not let me build the provided project.</span></strong></p><p>The project has been built in both 32/64 bit mode. If you are working on 32 bits, you might need the x64 compiler for Visual Studio 2010 to build the project.</p><p>&nbsp;</p><p><span style="font-size: 14px;"><strong>3. Building the project, I get linker error messages.</strong></span></p><p>Please go to the project properties and check the library settings. CUDA libraries have different names for x32 / x64. Make sure that the correct path and filenames are given.</p><p>&nbsp;</p><p><strong><span style="font-size: 14px;">4. I have built the project but the executables will not run (The application has failed to start because its side-by-side configuration is in<br />
correct.</span></strong><strong><span style="font-size: 14px;">)</span></strong></p><p>Please update the VS2010 redistributables to the PC you are running your executable and install all the latest patches for Visual Studio.</p><p>&nbsp;</p><p><span style="font-size: 14px;"><strong>5. My GPU-accelerated LIBSVM is running smoothly but i do not see any speed-up.</strong></span></p><p>GPU-accelerated LIBSVM is giving speed-ups mainly for big datasets. In the GPU-accelerated implementation some extra time is needed to load the data to the gpu memory. If the dataset is not big enough to give a significant performance gain, the gain is lost due to the gpu-memory -&gt; cpu-memory, cpu-memory -&gt; gpu-memory&nbsp; transfer time. Please refer to the graph above to have a better understanding of the performance gain for different dataset sizes.</p><p>Problems also seem to arise when the input dataset contains values with extreme differences (e.g. 10<sup>7</sup>) if <strong>no</strong> scaling is performed. Such an example is the &quot;breast-cancer&quot; dataset provided in the official LIBSVM page.</p><p>&nbsp;</p><p><u><font face="Arial" size="2"><span lang="EN-GB" style="font-size: 10pt; font-family: Arial;">ACKNOWLEDGEMENTS</span></font></u></p><p><font face="Arial" size="2"><span lang="EN-GB" style="font-size: 10pt; font-family: Arial;">This work was supported by the EU FP7 projects GLOCAL (FP7-248984) and WeKnowIt (FP7-215453).</span></font></p><p>&nbsp;</p><p><em>Feedback is kindly appreciated.</em></p>
